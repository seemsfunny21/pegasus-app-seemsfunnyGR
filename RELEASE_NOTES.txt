PEGASUS 237 - Morning Report Full Backup

Problem fixed:
- The user wants the daily morning report to also send a complete backup of PEGASUS data so future data loss can be recovered from the email.

Changes:
- Added PEGASUS 237 Daily Backup Collector in backup.js.
- Morning report now builds a fresh backup at send time, not the previous evening.
- Backup is injected into existing EmailJS template data through nutrition_advice so it is sent even if the EmailJS template has not been edited to add a new backup field.
- Additional params are also filled: backup_summary, backup_json, pegasus_backup_json, backup_schema, backup_key_count, backup_size_bytes, backup_truncated.
- Backup includes PEGASUS user data keys, food logs, cardio logs, weight keys, Kouki keys, parking, biometrics, lifting, car/service data, finance, notes, supplies, movies, YouTube, missions, social/contacts where present.
- Backup deliberately skips PIN/master/API/vault secrets, runtime logs, sync traces, old backups/snapshots and permanent-cache markers to avoid unsafe leaks and recursive oversized emails.
- Manual export filename is now PEGASUS_FULL_BACKUP_YYYY-MM-DD.json and uses the same complete user-data collector.
- Permanent local cache bumped to v237.

Install note:
Upload all files. First open after this package will build/check v237 local cache once. The next morning report should log: "PEGASUS REPORT: Daily backup attached" before sending.

Package:
- Code-only.
- No images folder.
- No videos folder.
