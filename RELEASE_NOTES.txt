PEGASUS 236 - Data Guard Repair Push + Fast Local Cache

Problem fixed:
- If JSONBin is overwritten manually with a thin payload, PEGASUS must not treat missing cloud fields as deleted data.
- After a safe merge from a thin cloud payload, PEGASUS now arms a full repair push so the cloud is rebuilt from the local full snapshot.
- Refresh was slow because permanent local storage was being checked/started on every open as if files needed to download again.

Changes:
- Added Data Guard repair marker: thin/partial cloud payloads trigger a safe full re-push after preserving local data.
- Missing remote keys still never delete local data unless an explicit safe delete channel is used.
- Critical non-empty local modules are preserved against empty remote values.
- Push clears the Data Guard repair marker only after successful full cloud write.
- Permanent local cache bumped to v236.
- Desktop loader now skips the download pipeline when v236 assets are already marked ready locally.
- Service worker stores a v236 cache metadata marker and returns 100% immediately on later starts.
- Mobile no longer forces service-worker update checks on every open.

Install note:
Upload all files. The first open after this package will rebuild the v236 local cache once. After it reaches 100%, normal refreshes should be much faster and should not re-download everything.

Package:
- Code-only.
- No images folder.
- No videos folder.
