PEGASUS 235 - Cloud Data Loss Guard

Problem fixed:
- A manual/thin JSONBin overwrite could contain only a few fields and omit most PEGASUS keys.
- The old pull logic treated missing remote keys as deletion and could wipe local modules like Ύπνος, Parking, Βάρη, Notes, Finance, YouTube, Supplies, etc.

Changes:
- Cloud pull is now merge-first and non-destructive: missing remote keys no longer delete local data.
- Added Data Guard snapshots before every cloud pull: latest / previous / older local safety copies.
- Empty remote values for critical modules no longer overwrite non-empty local values.
- Thin/partial cloud payloads are detected and logged as merge-only.
- Legacy/manual payload fields are mapped safely, e.g. car_service -> pegasus_car_service, kcal/protein/supp_inventory/weekly_history/food_library/contacts.
- Push now includes data-guard metadata and aborts if a bug creates an abnormally tiny payload.
- Added pegasus_weight_ and pegasus_lifting_v1 sync coverage for mobile lifting aliases.
- Bumped service-worker cache to v235 so cloudSync.js refreshes cleanly.

Package:
- Code-only.
- No images folder.
- No videos folder.
